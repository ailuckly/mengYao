# 系统实现说明

## 1. 系统实现概述

本系统采用 Flask 构建 Web 应用，主流程为：

`图片上传 -> 后端接收 -> YOLOv11 模型推理 -> 结果可视化 -> 识别结果与建议返回`

系统默认接入真实模型权重：

- 默认权重：`model/checkpoints/best.pt`
- 推理模式：优先真实模型，缺失权重时自动退回 mock 模式

## 2. 已实现模块

### 2.1 前端展示模块

前端采用多页面 Web 结构，包含：

- 首页
- 识别页
- 历史记录页
- 知识库页
- 模型分析页

其中识别页完成图片上传、结果图展示和识别明细展示；历史记录页用于查看与删除历史记录；知识库页用于展示植物和病害类别说明；模型分析页用于展示实验结果与训练图表。

### 2.2 后端接口模块

后端已实现以下接口与路由：

- `GET /health`：服务健康检查
- `GET /labels`：获取支持识别的类别
- `POST /predict`：上传图片并返回识别结果
- `GET /api/history`：获取历史记录列表
- `GET /api/history/<id>`：获取单条识别详情
- `GET /api/knowledge`：获取知识库数据
- `GET /api/analysis`：获取模型分析摘要

### 2.3 推理模块

推理模块已具备：

- 模型权重自动加载
- 单张图片识别
- 识别框绘制与结果图输出
- 类别与建议映射
- 识别记录自动保存
- 权重缺失时的降级占位预测

### 2.4 历史记录模块

系统采用 SQLite 实现历史记录管理，主要功能包括：

- 识别结果自动写入本地数据库
- 历史记录列表查询
- 单条记录详情查看
- 历史记录删除

数据库文件默认位于：

- `records/app.db`

### 2.5 知识库与分析模块

系统在知识辅助与实验展示层面包括：

- 最终类别映射设计与养护建议映射
- 植物与病害静态知识库配置
- 模型训练结果摘要配置
- 训练曲线图和混淆矩阵图展示

实验结果摘要如下：

- 使用模型：`YOLOv11s`
- 最佳 epoch：`83`
- mAP50：`0.7279`
- mAP50-95：`0.6754`

## 3. 系统范围

本系统定位为一套“本地可演示版 Web 识别系统”，聚焦以下能力：

1. 常见家庭园艺植物类别识别
2. 典型叶部病害与异常辅助识别
3. 识别结果可视化展示
4. 历史记录管理
5. 静态知识库展示
6. 模型分析与实验结果展示
7. 基础养护建议返回
