# 系统实现说明

## 1. 当前实现范围

当前系统已完成最小可运行版本的代码骨架，主要用于打通“页面上传 -> 后端接收 -> 推理处理 -> 结果返回 -> 页面展示”的主流程。由于真实模型文件和训练数据尚未加入，当前版本同时保留了占位推理逻辑，便于先完成系统联调和界面展示。

## 2. 已实现模块

### 2.1 页面模块

当前前端实现了一个单页识别界面，包含：

- 系统标题与简介
- 图片上传表单
- 状态提示区
- 图片预览区
- 识别结果图区
- 结果文字信息区

### 2.2 接口模块

后端当前已实现：

- `GET /health`
- `GET /labels`
- `POST /predict`

### 2.3 推理模块

当前推理模块支持两种运行方式：

- 若存在真实模型权重，则走真实推理逻辑。
- 若不存在模型权重，则走占位推理逻辑，返回一个可视化识别结果，便于前后端联调。

### 2.4 配置模块

当前系统已提供类别和养护建议配置文件：

- `model/configs/label_advice.json`

## 3. 当前版本的作用

当前版本不等同于最终毕业设计成品，但具备两个重要作用：

1. 提前确定系统接口、目录结构和模块边界。
2. 为后续接入真实 YOLOv11 模型提供固定入口，减少返工。

## 4. 后续补充方向

后续应继续完善以下内容：

- 增加真实训练数据集。
- 补充 YOLOv11 模型权重。
- 调整预测逻辑以匹配真实类别。
- 完善前端结果展示样式。
- 增加测试记录和系统截图。

## 5. 本章小结

本章说明了当前系统代码实现的阶段性成果。虽然真实模型和数据尚未接入，但最小系统骨架已经搭建完成，可以作为后续继续开发和论文截图的基础。
