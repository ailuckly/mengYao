# 基于YOLOv11的家庭园艺植物识别系统设计与实现

## 摘要

随着生活水平的提高和居住环境观念的变化，家庭园艺逐渐从单纯的兴趣爱好发展为一种兼具审美、休闲和情绪调节功能的生活方式。越来越多的家庭开始在阳台、客厅、书房等空间中摆放绿植或种植小型观赏植物。然而，普通用户通常缺乏系统的植物分类知识和病害识别经验，当植物出现黄叶、斑点、霉变、枯萎等异常现象时，往往只能依赖个人经验或者网络零散信息进行判断，导致处理不及时、处理方式不准确，甚至进一步影响植物生长状态。基于此，设计并实现一套面向家庭园艺场景的植物识别系统，利用计算机视觉技术完成植物类别识别和典型叶部异常辅助分析，具有一定的应用价值和现实意义。

本文围绕“基于YOLOv11的家庭园艺植物识别系统设计与实现”这一课题展开研究，结合公开数据集、目标检测模型和 Web 应用开发技术，完成了一个可运行、可展示、可用于毕业设计答辩的家庭园艺植物识别系统。系统总体采用“前端页面 + Flask 后端服务 + YOLOv11 检测模型”的实现方案。前端负责页面展示、图片上传、结果可视化和交互引导；后端负责接口组织、文件校验、推理调用、结果处理、历史记录管理和静态知识服务；模型部分则基于 YOLOv11 完成植物类别和叶部病害、异常类别的联合检测任务。

在数据构建方面，本文以公开数据集为主，结合场景适配和类别映射策略，构建了适合本课题的训练数据。植物类别主要来源于室内植物公开数据集，病害类别主要来源于 PlantDoc 等真实场景病害数据。针对题目“家庭园艺植物识别系统”的定位，本文没有直接保留原始公开数据中的全部细粒度类别，而是按照“少类别、差异明显、便于训练和展示”的原则，将最终识别目标划分为 10 类，其中包含 6 类家庭园艺植物和 4 类病害、异常类别。数据整理完成后，构建得到最终训练集 `final_v1`，其中训练集 7011 张、验证集 854 张、测试集 848 张，为模型训练和实验分析提供了统一数据基础。

在模型训练方面，本文采用 YOLOv11s 作为正式实验模型，基于 PyTorch 与 Ultralytics 训练框架完成训练与验证。训练过程中输入尺寸设置为 640，批量大小设置为 32，训练轮次为 100，采用自动优化器配置、余弦学习率策略以及多种增强方式。实验结果表明，系统最终训练得到的最佳模型出现在第 83 个 epoch，其 Precision 为 0.7441，Recall 为 0.7495，mAP50 为 0.7279，mAP50-95 为 0.6754。从类别表现来看，植物类别整体识别效果较好，能够满足系统演示和毕业设计论文分析需求；病害与异常类别整体弱于植物类别，但仍具备辅助判断能力。最终，系统选用最佳权重 `best.pt` 作为正式演示模型接入 Web 应用。

在系统实现方面，本文完成了首页、识别页、历史记录页、知识库页和模型分析页五个主要页面，并实现了图片上传、结果可视化、历史记录保存、知识卡片展示、训练结果分析等功能。为了增强系统完整度，本文在传统“上传图片并返回结果”的基础上，进一步加入 SQLite 历史记录管理模块、植物与病害知识库模块以及模型分析模块，使系统不仅能够完成识别，还能够展示结果追溯、辅助说明和实验支撑信息。测试结果表明，系统页面访问正常，识别主流程完整，关键接口稳定，自动化测试全部通过，能够满足毕业设计系统实现与展示要求。

本文的主要工作包括：完成家庭园艺植物识别系统的需求分析与总体设计；完成训练数据集的类别映射与整理；完成基于 YOLOv11s 的植物识别模型训练与实验分析；完成基于 Flask 的多页面 Web 系统实现；完成系统测试与实验结果整理。研究表明，采用 YOLOv11 结合 Web 系统设计的方法，能够较好地支撑家庭园艺植物识别这一场景化应用任务。该课题既体现了目标检测模型在细分视觉任务中的实际应用，也体现了人工智能专业毕业设计在工程实现、实验分析和系统展示方面的综合实践特征。

**关键词：** YOLOv11；家庭园艺；植物识别；病害检测；Flask；目标检测

## Abstract

With the improvement of living standards and the growing popularity of home gardening, more and more users are keeping ornamental plants in indoor environments such as balconies, living rooms and study rooms. However, ordinary users usually lack professional knowledge of plant classification and disease identification. When leaves show symptoms such as yellowing, spotting, mold, or wilting, it is difficult for them to make accurate judgments and take appropriate care measures. Therefore, it is meaningful to design a plant recognition system for home gardening scenarios and provide auxiliary analysis through computer vision technology.

This thesis focuses on the design and implementation of a home gardening plant recognition system based on YOLOv11. The system integrates public datasets, object detection models and Web application development technologies, and finally forms a deployable and demonstrable graduation project system. The overall architecture adopts a three-layer structure composed of front-end pages, Flask back-end services and a YOLOv11 detection model. The front end is responsible for image upload, result visualization and user interaction. The back end handles request processing, file validation, model invocation, result formatting, history record management and static knowledge services. The model layer performs joint detection of plant categories and typical leaf diseases or abnormalities.

For dataset construction, this thesis mainly relies on public datasets and reorganizes them according to the needs of the home gardening scenario. Plant categories are mainly derived from indoor plant datasets, while disease and abnormality categories are mainly derived from real-scene plant disease datasets. Instead of directly preserving all fine-grained source categories, the final recognition targets are merged into ten categories, including six plant categories and four disease or abnormality categories. After data cleaning, mapping and annotation unification, the final dataset `final_v1` is obtained, containing 7011 training images, 854 validation images and 848 test images.

For model training, YOLOv11s is selected as the formal experiment model. The training is conducted with an input size of 640, batch size of 32, 100 epochs, automatic optimizer selection and cosine learning rate scheduling. The best model appears at epoch 83, achieving a Precision of 0.7441, Recall of 0.7495, mAP50 of 0.7279 and mAP50-95 of 0.6754. Experimental analysis shows that the model performs well on plant categories and can provide acceptable auxiliary judgment for disease and abnormality categories. Therefore, the best weight file is used as the default demonstration model in the final system.

In terms of system implementation, the thesis completes five major pages: home page, detection page, history page, knowledge base page and model analysis page. The system supports image upload, recognition visualization, result traceability, local history storage, knowledge card display and training result presentation. Compared with a simple single-page demo, the final system has a clearer structure and stronger completeness, which is more suitable for thesis writing and defense demonstration. Functional testing, interface testing and stability testing indicate that the system runs correctly, the main workflow is complete, and all key automated tests pass successfully.

The main contributions of this thesis include requirement analysis and system design for a home gardening plant recognition system, dataset mapping and construction, model training and experiment analysis based on YOLOv11s, multi-page Web system implementation based on Flask, and integrated system testing and evaluation. The study shows that combining YOLOv11 with a lightweight Web application is an effective solution for home gardening plant recognition and auxiliary leaf abnormality analysis.

**Keywords:** YOLOv11; home gardening; plant recognition; disease detection; Flask; object detection

## 第1章 引言

### 1.1 研究背景

近年来，随着家庭生活品质的提升和居家审美需求的增强，植物在家庭空间中的使用越来越普遍。无论是传统的绿萝、吊兰、金钱树等观叶植物，还是兼具观赏与装饰效果的其他室内绿植，都在家庭环境中扮演着越来越重要的角色。家庭园艺不仅具有美化环境、净化空气的作用，还常常与情绪调节、休闲娱乐、空间氛围塑造等方面相关。尤其在年轻群体和城市居住人群中，家庭园艺已逐渐发展为一种较常见的生活方式。

但是，与专业种植者相比，普通家庭用户在植物识别和养护经验方面存在明显不足。很多用户在购买植物时只知道俗称，对其准确类别、习性和养护要点缺乏系统认识；在植物出现异常时，例如叶片发黄、出现斑点、边缘卷曲、表面发霉或整体萎蔫，也往往只能凭借直观印象进行判断。对于缺乏专业知识的用户而言，这些异常现象究竟源于病害、虫害、环境变化，还是浇水、通风、光照等养护问题，并不容易区分。错误判断会进一步影响处理方案，甚至造成植物持续恶化。

随着深度学习和计算机视觉技术的快速发展，图像识别和目标检测已经在农业监测、工业质检、医学影像、交通感知等领域取得了较好的应用效果。目标检测模型不仅能够完成类别判断，还能定位图像中目标的位置，适合一张图片中存在一个或多个植物目标的场景。相比传统人工查询方式，基于图像识别的植物识别系统能够降低使用门槛，提高信息获取效率，使普通用户通过上传图片即可获得初步判断结果。因此，面向家庭园艺场景设计一套植物识别系统，具有较强的实际意义。

从研究角度看，植物识别任务本身也具有一定复杂性。第一，不同植物之间存在形态相近的问题，尤其在观叶植物中，部分植物叶片颜色、形态和纹理差异较小，容易造成模型混淆。第二，同一植物在不同生长阶段、不同光照条件和不同拍摄角度下，图像表现会有明显变化。第三，病害和异常往往表现为局部颜色或纹理变化，目标小、边界不明显，识别难度较高。第四，家庭园艺场景中的背景复杂度通常高于实验室图像，拍摄过程中还会受到遮挡、光照变化、桌面背景、花盆结构等干扰因素影响。上述问题决定了该课题既不是简单的分类任务，也不是可以直接照搬通用数据集和通用界面的简单工程练习，而是一个需要综合考虑数据适配、模型训练和系统实现的综合性课题。

基于这些背景，本文选择以 YOLOv11 为核心模型，设计并实现一个面向家庭园艺场景的植物识别系统。系统不仅完成模型训练和目标检测任务，还实现前后端交互、结果展示、历史记录管理、植物知识辅助以及模型分析页面，从而形成一个结构完整、功能闭环清晰、适合论文写作和答辩演示的毕业设计作品。

### 1.2 研究意义

本课题的研究意义主要体现在应用价值、技术验证和工程实践三个层面。

首先，从应用层面看，家庭园艺植物识别系统能够为普通用户提供一种较为直观和低门槛的辅助工具。用户不需要掌握专业植物知识，只需上传一张图片，就能够获得植物类别或叶部异常的初步分析结果，并配合系统返回的简要建议开展后续观察和处理。尽管系统不能替代专业园艺诊断，但在家庭使用场景下具有较高的实用性。

其次，从技术层面看，植物识别和叶部异常检测是目标检测技术在细分视觉场景中的一个典型应用方向。通过构建适合家庭园艺场景的数据集、设计合理的类别映射方式并训练 YOLOv11 模型，可以验证目标检测技术在该场景下的适用性，为类似的小型智能识别系统提供一定的技术参考。

再次，从工程实践层面看，本课题并不仅停留在算法实验，而是完整经历了需求分析、方案设计、数据准备、模型训练、系统开发、测试验证和文档整理等全过程。这种“算法 + 系统”的实现方式能够较完整地体现人工智能专业毕业设计应有的综合性，既考察学生对目标检测模型和深度学习训练流程的掌握，也考察其在 Web 应用开发、系统设计和工程落地方面的能力。

### 1.3 国内外研究现状

#### 1.3.1 国外研究现状

国外在植物识别、病害检测以及通用目标检测方面起步较早，方法发展也相对系统。早期植物识别研究多采用传统图像处理和机器学习方法，通过人工提取颜色、纹理、形状等特征，再结合支持向量机、随机森林等分类器完成识别。这类方法在背景简单、目标单一的图像上能够获得一定效果，但当图像背景复杂、遮挡明显或者拍摄条件不稳定时，人工特征的鲁棒性较差，模型泛化能力有限。

随着卷积神经网络的发展，国外研究逐渐将深度学习引入植物识别和病害检测任务。深度学习模型能够自动从大量图像中学习高层语义特征，因此在复杂背景、多类别和细粒度识别任务中表现优于传统方法。特别是 YOLO 系列目标检测算法在实时性和精度之间取得了较好的平衡，逐渐成为一类重要的检测技术路线。YOLOv3 通过单阶段检测框架显著提高了检测速度，YOLOv4 和 YOLOv7 在主干网络、训练策略和特征融合等方面进一步优化，使实时目标检测精度持续提升。随着模型版本不断迭代，单阶段检测方法已经能够兼顾推理效率与识别效果。

在农业与植物识别应用中，国外大量研究面向农田、果园、温室等生产型场景开展，例如作物病斑检测、果实成熟度判断、病虫害识别和农业机器人视觉导航等。这些研究常使用卷积神经网络、目标检测网络甚至语义分割网络进行病叶定位、病害分类和作物区域提取。部分研究还引入注意力机制、多尺度特征融合、迁移学习等方法，以提升模型对小目标、复杂纹理和多类别病害的识别能力。整体来看，国外研究在算法先进性、实验对比和部署场景拓展方面具有较为成熟的基础。

不过，国外研究也存在一定局限。其数据场景大多面向农业生产环境，拍摄对象、拍摄角度和背景条件相对明确，和家庭园艺的真实使用场景仍有差异。家庭环境中的植物拍摄更随意，背景更杂乱，图像来源更加非标准化，因此已有国外研究虽然在方法上具有借鉴意义，但不能完全直接迁移到家庭园艺场景。

#### 1.3.2 国内研究现状

国内在植物识别与病害检测方向的研究近年来发展很快。随着 PyTorch、TensorFlow 和 Ultralytics 等工具的普及，越来越多研究开始围绕深度学习目标检测模型开展实验。在植物病害识别、作物病斑检测、叶片分类和农业智能监测系统方面，国内文献数量持续增加，研究内容也从单纯算法实验逐渐扩展到系统应用层面。

在方法选择上，国内研究普遍倾向于使用 YOLO 系列模型作为核心检测框架，并结合具体任务进行参数优化或结构改进。例如，有研究基于 YOLOv8 和其他单阶段检测模型开展植物叶片识别、病害定位和多类植物目标检测实验，通过改进网络结构、优化损失函数或增强训练策略，提高检测效果。也有研究从轻量化和部署角度出发，探索植物识别系统在移动端、边缘端和 Web 应用中的实际可行性。

在系统实现层面，国内很多研究开始关注“模型训练 + 系统实现”的完整工作流。常见方案是利用 Python、PyTorch 和 Flask 或 Django 构建后端服务，前端则通过 H5、Vue 或模板页实现上传和结果展示。这种方式使植物识别从实验环境扩展到可运行系统，提升了课题的工程属性和展示价值。

然而，从家庭园艺这一细分场景来看，国内研究仍然存在一些不足。第一，面向家庭环境的植物公开数据较少，许多研究依旧依赖农业病害数据或实验条件下拍摄的单目标图像，数据与实际使用场景之间存在差距。第二，一些研究更重视模型指标，而较少关注系统交互、结果解释和辅助建议等对普通用户更直接的内容。第三，病害识别在家庭场景中常面临样本不均衡、类别区分不明显等问题，使模型在异常类上的表现弱于植物类别本身。因此，面向家庭园艺场景设计完整系统仍具有进一步研究和实践价值。

#### 1.3.3 现有研究存在的问题

综合国内外研究可以发现，当前植物识别与病害检测领域主要存在以下问题。

第一，公开数据与家庭场景之间不完全匹配。很多数据集来自农业生产环境或实验采集环境，背景相对简单，植物种类和病害种类也更偏农作物，而家庭园艺场景中常见的是室内观叶植物、盆栽植物和小型观赏植物，两者在视觉表现和使用需求上存在差异。

第二，病害和异常类别的细粒度差异较小，数据不均衡问题明显。相较于植物种类识别，病害异常通常表现为叶片局部颜色变化、斑点、霉层或黄化区域，不仅区域较小，而且部分类别之间表征接近，容易造成误判和漏检。

第三，部分系统研究强调算法指标，但忽视系统完整性。仅有模型训练结果并不足以构成一套完整的毕业设计系统，仍需要前端页面、接口交互、结果可视化、记录管理和功能测试等环节，才能形成真正意义上的系统设计与实现。

第四，结果解释和用户服务能力不足。对家庭用户而言，仅返回类别名称和检测框并不能完全满足需求。如果系统能够进一步提供简要养护建议、异常说明和历史记录查看能力，实用性会更强。

### 1.4 研究内容

针对上述问题，本文围绕家庭园艺植物识别系统展开研究，主要完成以下内容。

1. 分析家庭园艺场景下植物识别系统的应用需求，完成需求分析、总体设计和详细设计。
2. 以公开数据集为基础，完成植物类别与病害、异常类别的筛选、映射与统一标注，构建适合本课题的训练数据。
3. 基于 YOLOv11s 完成模型训练与实验分析，评估模型在植物类别和叶部异常识别中的表现。

## 第2章 相关技术与方案论证

### 2.1 YOLOv11 目标检测技术

YOLO 系列算法是一类典型的单阶段目标检测算法，其核心思想是将目标检测视作一个端到端回归问题，在一次前向传播中同时完成目标位置回归和类别预测。与两阶段检测方法相比，YOLO 系列在推理速度方面具有明显优势，因此在实时检测、轻量部署和交互式应用中被广泛采用。

本文选择 YOLOv11 作为基础检测框架，主要原因如下。第一，YOLO 系列算法结构成熟，社区文档与工具支持较完善，适合毕业设计场景下的快速实验和工程落地。第二，YOLO 系列在目标定位和类别识别上具有较好的综合表现，可以同时完成“植物类别识别 + 病害区域检测”这一复合任务。第三，YOLOv11 与 Ultralytics 训练框架配合较好，方便进行权重加载、训练管理、预测调用和实验结果导出。

在本课题中，YOLOv11 的应用重点主要体现在以下几个方面：

1. 统一检测与分类流程。  
   家庭园艺识别系统既需要识别植物种类，也需要识别典型叶部病害、异常。如果采用传统分类模型，则需要手工裁剪或假设单主体图像；采用目标检测模型则能够直接对整图进行处理，更适合真实用户上传场景。

2. 支持单图多目标情况。  
   家庭用户上传的图片中可能同时包含多个叶片目标、多个异常区域或者植物主体与病叶区域。目标检测模型可以输出多个检测框和多个类别结果，为系统可视化展示提供更丰富信息。

3. 兼顾速度与效果。  
   家庭园艺识别系统面对的是交互式应用而非离线批处理任务。用户更关注“上传后能否快速得到结果”，因此算法在保证基本精度的同时，还需要维持较好的推理效率。YOLOv11s 在模型规模、检测性能和部署便利性之间取得了较好的平衡，适合作为正式演示模型。

### 2.2 PyTorch 深度学习框架

PyTorch 是当前深度学习研究和工程中应用广泛的框架之一，具有动态图机制灵活、调试方便、社区活跃以及与 Python 生态兼容性强等优点。对于毕业设计而言，PyTorch 的优势主要体现在训练流程直观、模型调用便捷以及与现有视觉工具链兼容程度高。

本文采用 PyTorch 完成模型训练与推理，主要基于以下考虑：

1. PyTorch 与 Ultralytics 框架结合紧密，便于直接进行 YOLO 模型训练、验证和推理。
2. Python 生态完善，便于数据处理、图像操作、结果记录和实验统计。
3. PyTorch 在 Colab、云 GPU 和本地环境中均有较成熟的支持，适合学生在有限硬件条件下开展实验。

### 2.3 Flask Web 框架

Flask 是一种轻量级 Python Web 框架，结构清晰，学习成本较低，适合中小型项目快速开发。与大型框架相比，Flask 在路由组织、模板渲染、静态资源管理和接口开发方面都较为灵活，更适合本课题这种既需要页面展示又需要后端推理调用的系统。

本文采用 Flask 而不选择更复杂的 Web 框架，主要原因如下：

1. 本课题系统规模适中，不涉及复杂用户体系和后台权限管理，使用 Flask 可以减少不必要的框架负担。
2. Flask 能够同时支持模板渲染和 JSON 接口，便于实现“页面访问 + 异步识别请求”的混合模式。
3. Flask 与 Python 推理服务集成自然，可直接在同一工程中组织模型调用、结果处理和页面返回逻辑。

### 2.4 SQLite 本地数据库

虽然系统早期设计中曾以文件存储为主，但在实际实现过程中，为了增强系统完整性和可追溯性，本文最终引入 SQLite 作为历史记录存储方案。SQLite 具有零配置、轻量、便于本地部署和单文件管理的特点，非常适合毕业设计项目中“无需复杂数据库服务器但需要一定结构化存储”的场景。

在本系统中，SQLite 的主要作用是保存识别历史记录，包括识别时间、图片名称、主判定类别、结果图路径、建议内容和原始预测结果等信息。与纯文件保存相比，SQLite 便于进行列表查询、详情查看和删除操作，使系统从“演示页”提升为“具有结果管理能力的完整应用”。

### 2.5 前端实现技术

前端部分采用 Flask 模板、HTML5、CSS 和原生 JavaScript 实现，不引入复杂的前后端分离框架。这样做的原因主要有三点。

第一，本课题重点在于模型识别与系统整合，而不是前端工程化框架本身。采用模板页面可以降低开发复杂度，把更多精力用于结果展示、页面组织和交互细节。

第二，多页面结构更符合毕业设计演示需求。首页、识别页、历史记录页、知识库页和模型分析页可以分别承载不同功能，整体观感也更接近完整系统。

第三，原生 JavaScript 足以完成文件上传、图片预览、接口调用和结果更新等交互，且维护成本低，更适合论文中对系统实现逻辑进行说明。

### 2.6 技术方案论证

为了保证系统定位清晰、工作量合理且能够在规定时间内完成，本文对若干技术路径进行了比较和论证。

#### 2.6.1 分类模型与检测模型的比较

如果仅采用图像分类模型，系统实现会相对简单，但这类模型通常假设整张图片只对应一个主体类别，难以自然处理真实拍摄中可能出现的多目标、多区域异常等情况。而目标检测模型既能够输出类别，又能返回检测框，更适合系统页面中的结果图可视化需求。因此，本文最终采用检测模型作为统一识别框架。

#### 2.6.2 YOLOv11n 与 YOLOv11s 的比较

在实验过程中，轻量模型可以更快完成训练和验证，但从最终系统演示和论文结果展示来看，YOLOv11s 在精度与稳定性方面更具优势。结合当前训练结果，本文最终选用 YOLOv11s 作为正式模型权重，对外展示最佳实验结果。

#### 2.6.3 文件记录与数据库记录的比较

如果完全采用文件方式记录结果，系统虽然也能运行，但历史记录查询、详情查看和删除操作都需要额外处理，系统完整度较弱。引入 SQLite 后，系统可以更自然地形成“识别 -> 存储 -> 查询 -> 删除”的闭环流程，因此本文最终采用“文件存储图像 + SQLite 保存结构化记录”的方式。

#### 2.6.4 单页系统与多页面系统的比较

单页上传系统开发难度较低，但页面层次单一，工作量展示有限，不利于毕业设计答辩和论文截图整理。多页面结构虽然需要额外设计导航和页面组织，但能够更好地体现系统完整性，也便于将不同功能映射到不同章节。因此，本文最终采用多页面结构。

### 2.7 本章小结

本章介绍了本文使用的主要相关技术，并对检测模型、训练框架、Web 后端、数据库方案和前端实现方式进行了论证。综合比较后，本文最终采用“YOLOv11s + PyTorch + Flask + SQLite + 多页面模板系统”的技术路线。这一路线既能够满足植物识别系统的功能需求，又能与毕业设计的工作量、开发周期和展示需求相匹配。

本系统以 Python 为主要开发语言，以 PyTorch 和 Ultralytics 为训练与推理基础，以 Flask 为 Web 服务框架，整体技术路线较为成熟。YOLO 系列模型在目标检测领域具有较强的工程实用性，适合本课题这种“识别 + 定位 + 可视化”的任务形式。前端采用模板页面与原生 JavaScript，能够较好地满足本课题规模下的交互需求。综合来看，本课题在技术实现上可行。

本项目开发主要依赖开源框架和现有设备，无需额外商业授权。数据来源以公开数据集为主，模型训练也可通过 Colab 等云 GPU 完成，整体开发成本较低，适合本科毕业设计的条件。

### 3.3 业务需求分析

从实际使用场景来看，用户通常会在植物出现异常、想了解植物名称，或者需要对植物健康状况进行初步判断时使用系统。因此，系统需要满足以下业务需求：

1. 用户可以快速完成图片上传，不需要复杂注册和配置。
2. 系统能够返回植物类别或叶部异常类别，并提供基本说明。
3. 系统应把识别结果以直观方式呈现，方便用户理解。
4. 系统应保存历史记录，便于回看和对比。
5. 系统应提供一定的知识辅助，增强系统实用性和完整度。

基于上述业务需求，系统主流程可以描述为：

用户进入系统首页 -> 前往识别页上传图片 -> 系统完成校验和模型推理 -> 页面展示识别结果和建议 -> 系统自动保存历史记录 -> 用户可进一步查看知识库或模型分析页。

### 3.4 功能需求分析

#### 3.4.1 图片上传功能

系统需要支持 JPG、JPEG、PNG 等常见图片格式的上传，并在前端展示本地预览效果。上传阶段需要进行扩展名校验和图片合法性检查，对非法文件进行拦截并返回明确提示。

#### 3.4.2 植物类别识别功能

系统应对训练集中定义的植物类别进行识别。当前正式系统支持 6 类家庭园艺植物，包括银皇后类、网纹凤梨类、绿萝类、心叶蔓绿绒类、Rhaphidophora 类和金钱树类。对于输入图片中的目标植物，系统需要返回类别名称、置信度和结果图。

#### 3.4.3 病害与异常辅助识别功能

系统应对典型叶部病害和异常进行辅助识别。当前正式系统支持 4 类异常目标，包括叶斑类病害、枯萎/疫病类病害、白粉/霉变类病害和黄化/锈病/虫害类异常。该功能的定位不是专业诊断，而是为用户提供初步判断。

#### 3.4.4 结果展示功能

识别完成后，系统应展示原图、结果图、主判定类别、置信度、推理来源以及按检测项组织的结果详情，使用户能够直观看到系统判断依据。

#### 3.4.5 历史记录功能

系统需自动保存每次识别结果，并支持历史记录列表查看、详情查看和删除操作。该功能既增强系统完整度，也便于答辩展示和论文截图整理。

#### 3.4.6 知识库展示功能

系统应围绕最终 10 个类别提供静态知识卡片，包含类别别名、典型特征、养护或处理建议以及风险提示等内容，用于增强系统的辅助服务能力。

#### 3.4.7 模型分析功能

系统应提供一页集中展示训练模型、最佳 epoch、关键指标、训练曲线和混淆矩阵，用于支撑论文中的实验分析章节。

### 3.5 非功能需求分析

#### 3.5.1 性能需求

系统对性能的要求主要集中在响应时间和稳定性。对单张图片识别而言，应在合理时间内完成上传、推理和页面返回；对于答辩演示而言，系统更强调交互流畅性和结果稳定性，而不是极端实时性指标。

#### 3.5.2 易用性需求

页面结构应清晰，上传入口明显，识别结果重点突出，错误提示简洁易懂。普通用户不需要了解模型原理，只需知道如何上传图片和查看结果即可。

虽然本系统定位于毕业设计实现版本，但结构设计上仍需保留一定扩展能力，例如未来增加类别、增加批量识别、增加更复杂的历史统计等功能时，不应对现有结构造成大范围破坏。

## 第6章 模型训练与识别实现

### 6.1 数据集来源与类别设计

本课题训练数据以公开数据集为主，不再采用“自采图像为主”的方案，而是通过公开数据的筛选、重组和统一标注来构建适合家庭园艺场景的训练数据。这样既符合当前开发条件，也能够保证数据来源清晰，便于论文表达。

植物类别主要来源于室内植物公开数据集，病害和异常类别主要来源于 PlantDoc 等真实场景病害数据。考虑到题目是“家庭园艺植物识别系统”，若直接沿用源数据中的全部细粒度类别，不仅训练难度大，而且系统定位会变得模糊。因此，本文按照“少类别、差异明显、兼顾实用性与可训练性”的原则，对原始数据进行了类别映射和筛选。

最终系统支持 10 类目标，具体如下：

### 6.2 数据清洗与标注规范

为了保证训练质量，本文在数据整理过程中遵循统一的数据清洗与标注规范。主要原则包括：

1. 删除重复样本和明显不可用样本；
2. 删除严重模糊、曝光异常或完全失焦图像；
3. 对类别冲突样本进行人工复核；
4. 保持类别编号和类别命名的一致性；
5. 标注框尽量紧贴目标区域，避免过度偏移。

对于植物类别数据，由于部分源数据为单主体植物图像，因此在训练准备阶段将其转换为适合检测任务使用的整图或主体区域框格式；对于病害数据，则尽量保留原始检测框信息，以增强模型对异常区域的学习能力。

### 6.3 数据集划分与规模

经过整理后的正式训练数据版本为 `final_v1`，其数据集划分情况如下：

- 训练集：7011 张
- 验证集：854 张
- 测试集：848 张

这种划分方式保证了训练、验证和测试之间的基本独立性，也便于后续在系统中固定使用验证结果和测试结果进行分析。

### 6.4 训练环境与训练脚本

本课题模型训练主要基于 PyTorch 和 Ultralytics 框架完成。训练环境可在本地或云 GPU 环境中运行，实际正式训练采用 Colab Pro 环境完成，以解决本地无 GPU 的限制。

项目中已补充完整的训练支持脚本，包括：

- `train.py`：用于正式训练和断点续训；
