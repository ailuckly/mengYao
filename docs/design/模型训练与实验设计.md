# 模型训练与实验设计

## 1. 设计目标

本章用于说明系统中 YOLOv11 模型的训练思路、数据来源、实验设置和评估方式，为后续真正开展模型训练和撰写毕业设计说明书提供统一框架。

## 2. 数据集设计

### 2.1 数据来源

系统训练数据拟来自三个部分：

1. 公开植物图像数据集。
2. 网络补充图像。
3. 自采家庭园艺场景图像。

### 2.2 数据内容

数据集中应包括：

- 常见家庭园艺植物类别。
- 常见病害类别。
- 多种背景条件。
- 多种光照条件。
- 单目标和多目标样本。

### 2.3 数据划分

训练集和测试集按 8:2 比例划分，必要时可额外划出验证集。划分时尽量保证各类别样本分布相对均衡。

### 2.4 目录约定

当前项目已固定采用 `data/splits/train`、`data/splits/val`、`data/splits/test` 的目录结构，并配套 `model/configs/dataset.example.yaml` 作为训练入口配置。后续只需按该结构放入图片和标签即可开始训练。

## 3. 训练环境设计

建议训练环境如下：

- Python 3.10 或以上
- PyTorch
- Ultralytics / YOLOv11
- GPU 环境优先，如条件不足可使用 CPU 做小规模验证

## 4. 训练参数设计

初始训练参数建议作为基线配置：

- 输入尺寸：640
- 批大小：16
- 学习率：0.01
- 训练轮数：100
- 优化器：SGD 或 AdamW
- 损失函数：采用 YOLO 默认配置，并结合 CIoU 相关策略

这些参数后续可根据实验结果调整。

## 5. 数据增强设计

为了提高模型在家庭园艺场景中的泛化能力，训练时可考虑使用以下数据增强方式：

- Mosaic 增强
- 随机翻转
- 随机亮度调整
- 随机缩放
- 随机裁剪

增强策略应以贴近真实拍摄场景为原则，避免生成明显不合理的样本。

## 6. 实验设计

### 6.1 基线实验

首先训练一个基线模型，用于观察系统在当前数据集上的基本识别效果。

### 6.2 参数对比实验

在基线模型基础上，逐项调整训练参数，例如输入尺寸、训练轮数、学习率和增强策略，并记录实验结果。

### 6.3 推理速度测试

在模型精度之外，还需记录单张图片推理耗时，以验证系统是否满足实时性要求。

## 7. 评估指标设计

系统主要采用以下指标进行评估：

- Precision
- Recall
- mAP
- F1-score
- 单张图片推理时间

## 8. 实验记录要求

每轮实验至少记录以下内容：

- 数据集版本
- 类别范围
- 训练参数
- 训练轮数
- 最优指标
- 典型识别结果截图
- 问题分析
