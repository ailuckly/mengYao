# 系统详细设计

## 1. 设计目标

系统详细设计的目标是在总体设计基础上，把系统的核心模块拆分到可编码实现的粒度，明确前端、后端、推理模块、结果处理模块和文件存储模块之间的关系，为后续系统开发和论文撰写提供统一依据。

## 2. 前端详细设计

### 2.1 页面组成

系统前端采用 H5 页面实现，当前版本包含一个主页面，集成了图片上传、图片预览和识别结果展示三个区域。后续如果需要扩展，可以再拆分为首页、识别页和说明页。

### 2.2 页面功能

- 选择本地图片文件。
- 预览待识别图片。
- 提交识别请求。
- 显示识别状态。
- 展示识别结果图。
- 展示类别、置信度和养护建议。

### 2.3 前端处理流程

1. 用户选择图片。
2. 页面调用本地预览逻辑显示图片。
3. 用户点击识别按钮后，页面通过 `fetch` 方式向 `/predict` 发送 `FormData`。
4. 接收到后端返回的 JSON 数据后，页面渲染识别结果。
5. 若接口返回异常，则展示错误提示。

## 3. 后端详细设计

### 3.1 后端结构

后端采用 Flask 实现，按职责拆为以下几层：

- 路由层：接收请求并返回响应。
- 服务层：组织业务流程。
- 推理层：执行模型预测。
- 工具层：处理文件校验、上传保存和建议映射。

### 3.2 路由设计

当前系统包含三个核心接口：

- `GET /health`
- `GET /labels`
- `POST /predict`

### 3.3 业务处理流程

后端识别流程如下：

1. 接收上传图片。
2. 校验图片文件名和扩展名。
3. 将图片保存到 `records/uploads/`。
4. 调用推理模块读取图片并执行预测。
5. 将预测类别映射为文字标签。
6. 根据类别匹配养护建议。
7. 生成结果图片并保存到 `records/predictions/`。
8. 返回结构化 JSON 数据。

## 4. 推理模块详细设计

### 4.1 推理模块职责

推理模块负责统一封装模型加载和预测逻辑，避免把模型细节直接写在路由中。当前采用 `YOLOPredictor` 类作为统一入口。

### 4.2 推理模式

为了兼顾当前开发进度和后续模型接入，推理模块支持两种模式：

1. 真实模型模式  
   当 `model/checkpoints/best.pt` 存在且本地安装了 `ultralytics` 时，直接加载 YOLO 模型执行预测。

2. 占位演示模式  
   当模型文件尚未准备好时，使用占位预测逻辑返回一个可视化结果，保证前后端链路可运行。

### 4.3 模型输出格式

推理模块输出统一采用列表结构，每一项包含：

- `label`
- `score`
- `box`
- `result_image`

这种结构便于前端直接渲染，也方便写入论文中的接口设计部分。

## 5. 结果处理模块设计

### 5.1 处理内容

模型原始输出需要进一步处理后才能展示给用户，主要包括：

- 类别编号映射为中文名称。
- 置信度整理。
- 检测框绘制到结果图。
- 匹配基础养护建议。

### 5.2 建议映射设计

建议映射通过本地配置文件 `model/configs/label_advice.json` 管理。配置中同时保存类别标签和建议文本，便于后续替换和扩展。

## 6. 文件存储与记录模块设计

### 6.1 上传文件存储

用户上传的原始图片保存到 `records/uploads/` 目录，用于调试、测试和系统演示。

### 6.2 结果文件存储

识别完成后的结果图保存到 `records/predictions/` 目录，用于页面展示和论文截图。

### 6.3 日志记录

系统运行日志预留在 `records/logs/` 目录，后续可以根据需要增加访问日志和异常日志。

## 7. 关键类与关键文件说明

- `backend/app.py`：应用启动入口，负责注册路由和页面模板。
- `backend/routes/predict.py`：定义系统接口。
- `backend/services/plant_service.py`：定义识别业务流程。
- `backend/inference/predictor.py`：封装模型加载与预测逻辑。
- `backend/utils/files.py`：处理上传文件保存。
- `backend/utils/advice.py`：处理类别与建议映射。

## 8. 本章小结

本章在总体设计基础上，对系统前端、后端、推理模块、结果处理模块和文件存储模块进行了细化说明，已经形成了可直接对应代码实现的设计方案。后续开发时，将按照本章划分的模块边界逐步完成具体功能。
