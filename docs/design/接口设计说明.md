# 接口设计说明

## 1. 接口设计目标

系统接口设计的目标是为前端页面、后端服务和后续真实模型接入提供统一的数据交换格式，避免后续功能增加时频繁修改请求和响应结构。

## 2. 接口清单

### 2.1 `GET /health`

用于检查服务状态。

返回字段：

- `status`
- `message`
- `inference_mode`
- `model_loaded`

### 2.2 `GET /labels`

用于返回当前系统支持的植物和病害类别列表。

返回字段：

- `status`
- `labels`
- `inference_mode`

### 2.3 `POST /predict`

用于上传图片并获取识别结果。

请求参数：

- `file`：图片文件

返回字段：

- `status`
- `message`
- `image_name`
- `labels`
- `scores`
- `boxes`
- `advice`
- `result_image`
- `predictions`
- `meta`

## 3. `predictions` 字段说明

`predictions` 为列表结构，单项包含：

- `label`
- `score`
- `box`
- `result_image`
- `source`

## 4. `meta` 字段说明

`meta` 为本次识别的附加信息，当前包含：

- `inference_mode`
- `prediction_count`
- `model_loaded`

## 5. 错误响应设计

当请求失败时，系统返回统一结构：

- `status: error`
- `message: 错误提示信息`

## 6. 本章小结

当前接口设计已经能够满足最小系统演示、前后端联调和后续接入真实模型的需求。后续如增加历史记录或用户功能，可在保持现有接口不变的前提下继续扩展。
