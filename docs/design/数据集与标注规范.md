# 数据集与标注规范

## 1. 规范目标

为了保证后续模型训练效果和实验记录一致性，数据采集、清洗、命名和标注需要遵循统一规范。该规范同时服务于系统实现和毕业设计说明书中的“数据集构建”章节。

## 2. 数据采集规范

### 2.1 植物类别图片

- 每个植物类别尽量覆盖不同角度、不同距离和不同背景。
- 尽量避免全部使用纯净背景图片。
- 同类植物至少保留一部分室内自然场景图。

### 2.2 病害类别图片

- 病害样本尽量包含轻度、中度两种情况。
- 同一病害尽量覆盖不同叶片大小和不同拍摄角度。
- 模糊严重、遮挡严重样本可保留少量作为鲁棒性测试样本，但不宜过多。

## 3. 数据清洗规范

- 删除重复图片。
- 删除过度模糊、严重曝光或完全失焦图片。
- 删除无法判断类别的样本。
- 对类别冲突图片进行人工复核。

## 4. 文件命名规范

建议采用以下命名格式：

```text
类别名_编号.jpg
病害名_编号.jpg
```

例如：

- `pothos_001.jpg`
- `rose_023.jpg`
- `leafspot_015.jpg`

## 5. 标注规范

### 5.1 标注原则

- 标注框应尽量紧贴目标边缘。
- 若一张图有多个目标，应分别标注。
- 若目标遮挡严重但仍可辨识，可保留标注。
- 无法确认类别时不要强行标注。

### 5.2 类别一致性

- 一个类别只能对应一个统一编号。
- 训练前应固定类别表，不在训练过程中频繁改编号。

## 6. 数据划分规范

- 训练集、验证集、测试集应按 8:1:1 或 8:2 方式划分。
- 同一拍摄批次的高度相似图片不要全部落在同一集合。
- 每类数据在验证集和测试集中都应有基本覆盖。

## 7. 本章小结

通过统一数据集和标注规范，可以减少训练阶段因样本质量不一致带来的问题，也方便后续把数据处理过程写入毕业设计说明书。
