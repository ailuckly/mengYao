## 1. 文献综述

### 一、课题背景与意义

随着生活水平的提高，越来越多的人开始在家里养花、种菜或者布置小型园艺空间。家庭园艺已经不只是简单的兴趣爱好，它还和居家环境、美化生活、调节情绪等方面有关。但在实际养护过程中，很多普通用户并不具备系统的植物知识，遇到植物叶片发黄、长斑、卷曲或者生长状态异常时，往往只能凭经验判断，识别不准确，处理也不及时。这样一来，不仅容易影响植物正常生长，还可能增加后续养护成本。

在这种背景下，如果能够借助计算机视觉技术，对家庭园艺植物的种类和常见病虫害进行自动识别，就可以在一定程度上帮助用户更快发现问题，减少盲目处理的情况。近几年，深度学习在图像识别领域发展很快，尤其是目标检测技术已经从实验室研究逐步走向实际应用，在农业识别、工业检测和安防监测等方向都取得了比较好的效果[1][2]。对于家庭园艺场景来说，这类方法同样具有较强的应用潜力。

从技术角度看，植物识别本身就是一个有一定难度的图像识别问题。不同植物之间有时外形接近，同一种植物在不同生长阶段又会表现出明显差异；病害的早期症状往往不够明显，只体现在局部颜色、纹理或者形状变化上[12]。另外，家庭环境中的拍摄条件比较复杂，背景杂乱、光照变化大、目标被遮挡等情况都很常见，这些因素都会影响识别结果。因此，研究适合家庭园艺场景的识别方法，既有一定的研究价值，也有很强的现实意义。

本课题选择基于YOLOv11来设计家庭园艺植物识别系统，主要有两方面考虑。一方面，YOLO系列算法一直以检测速度快、整体结构清晰、便于工程部署而受到广泛关注[2][5][8]。在需要兼顾实时性和准确率的应用中，这类算法比较适合。另一方面，家庭园艺识别系统最终不仅要做出模型，还要落到系统实现上，需要考虑图像上传、模型推理、结果展示和用户交互等问题。因此，本课题既包含算法研究，也包含系统开发，能够把人工智能专业所学的模型训练、系统设计和工程实现结合起来。

总体来看，本课题的意义主要体现在以下几个方面。第一，从应用层面看，它能够为普通家庭用户提供一个更直观的植物识别工具，帮助用户及时了解植物种类和健康状态，提高养护效率。第二，从研究层面看，可以进一步验证YOLOv11在家庭园艺这一细分场景下的适用性，为类似的植物识别任务提供参考。第三，从毕业设计的角度看，本课题涉及文献查阅、数据收集、模型训练、前后端开发、测试验证和文档撰写等多个环节，能够较全面地锻炼学生的综合实践能力。

### 二、国内外研究现状

#### （一）国外研究现状

国外对植物识别和目标检测的研究起步较早，相关方法的发展也比较系统。早期研究主要集中在传统图像处理和机器学习方法上，通常先对植物叶片的颜色、纹理、形状等特征进行人工提取，再利用支持向量机、决策树等模型完成分类。这类方法在背景简单、目标单一的情况下有一定效果，但当图像环境变复杂以后，特征稳定性和泛化能力都比较有限。

随着深度学习的发展，国外研究逐渐从传统特征工程转向卷积神经网络和目标检测模型。Redmon等提出YOLOv3后，单阶段目标检测方法在实时性方面表现突出，使“识别+定位”可以在一次前向传播中完成[2]。之后，YOLOv4在主干网络、特征融合和训练策略上进一步优化，整体检测效果得到提升[8]；YOLOv7则继续改善模型速度和精度之间的平衡，在实时检测任务中表现更强[5]。从技术路线来看，国外研究已经形成了比较明确的发展方向，即以深度学习为核心，通过更高效的特征提取网络、多尺度特征融合和更合理的损失函数设计，提高复杂场景下的检测性能。

在植物识别领域，国外研究者较早把深度学习用于作物病害识别、叶片分类、果实缺陷检测和农业机器人视觉感知等任务。研究中常用的理论模型主要包括卷积神经网络、目标检测模型和分割模型。相比传统方法，这些模型能够自动学习图像中的高层特征，因此在复杂背景下通常能取得更好的识别效果。部分研究还引入迁移学习、注意力机制和多尺度特征融合等方法，用来解决样本不足、小目标识别困难和细粒度分类精度不高等问题。整体来看，国外文献更强调模型性能提升、算法通用性以及农业智能化场景下的实际部署能力。

不过，国外研究更多集中在农田、果园、温室等农业生产环境，拍摄条件往往相对稳定，目标类别也较明确。相比之下，家庭园艺场景的图像来源更随意，背景更复杂，用户拍摄行为也不固定，因此国外已有研究成果虽然提供了较成熟的方法基础，但并不能完全直接适用于家庭园艺植物识别任务[3][13]。

#### （二）国内研究现状

国内在植物识别和病虫害检测方向的研究虽然起步相对晚一些，但近几年发展很快。随着深度学习框架和开源模型的普及，越来越多研究开始围绕植物图像识别开展实验。雷明等指出，深度学习方法在植物病虫害识别中已经显示出较明显优势，尤其在纹理特征提取和复杂病斑分类方面，比传统方法更有效[1]。这说明国内研究已经从简单分类逐步向更复杂的检测和识别任务扩展。

在主要方法上，国内很多研究以YOLO系列模型为核心开展改进。孙健等在YOLOv8基础上研究植物叶片识别问题，通过调整网络结构和训练策略来提高识别性能[6]。张慧等基于YOLO完成多类植物识别系统设计，也证明了目标检测方法在植物类别识别中的可行性[14]。从技术路线来看，国内研究一般采用“数据集构建或整理+模型训练优化+实验对比分析”的方式推进，一些研究还会结合数据增强、轻量化设计和部署测试，以提高模型在实际场景中的适用性。

除了算法层面，国内文献也逐渐开始关注家庭园艺这一具体应用场景。赵晓宇等总结了计算机视觉技术在家庭园艺植物分类中的应用情况，指出家庭环境中的复杂背景、近距离拍摄、局部遮挡和非标准化图像采集，是影响识别准确率的重要因素[3]。刘畅等对家庭园艺植物数据集构建方法进行了分析，认为仅依赖公开农业数据难以满足家庭场景需求，自建数据集对提升模型泛化能力很关键[9]。李雪等从图像扩充角度出发，说明适当的数据增强可以缓解样本不足问题，但增强方式需要与真实场景一致，否则容易造成训练偏差[16]。

在系统实现方面，国内研究通常采用Python和PyTorch作为训练基础，再结合Flask、Django等框架搭建识别系统[10]。王健等认为，模型压缩、推理加速和部署优化是影响系统落地效果的重要因素[15]。陈宇等围绕移动端植物识别系统开发开展研究，也说明当前研究已经不再只停留在算法层面，而是逐步转向“模型训练+系统实现+前端展示”的完整应用模式[17]。总体来看，国内研究已经具备一定基础，但面向家庭园艺场景的完整系统设计仍然不够多，尤其在结果解释和用户交互方面还有进一步完善的空间。

#### （三）最新进展与未来趋势

从最新进展来看，目标检测领域仍在持续更新，YOLO系列模型也在不断演进。周洋等认为，YOLOv11在结构设计和轻量化方面具有一定优势，更适合用于实时检测和边缘部署场景[11]。这说明当前研究正在从“单纯提高精度”转向“兼顾精度、速度和部署效率”的方向。对于家庭园艺植物识别来说，这种变化非常重要，因为实际应用更强调响应速度和使用便利性，而不只是实验环境下的单项指标。

结合现有文献可以看出，该领域未来大致会朝几个方向继续发展。第一，数据层面会更加重视场景化和细粒度化，也就是构建更贴近真实家庭环境的数据集，提高模型面对复杂背景和多类别目标时的适应能力。第二，方法层面会继续向轻量化、多尺度特征融合和鲁棒性优化发展，使模型在普通设备甚至移动端上也能保持较好的识别效果。第三，系统层面会更强调完整应用闭环，不仅要完成识别，还要结合前端交互、结果解释和养护建议输出，提高系统的实际可用性。第四，未来研究还可能更加注重多任务融合，例如把植物种类识别、病害检测和生长状态分析结合起来，形成更完整的智能园艺辅助系统。

总体来看，国内外研究已经为本课题提供了较成熟的方法基础和实现思路，但在家庭园艺这一具体场景下，仍然存在数据适配不足、系统化程度不够和结果服务能力较弱等问题。因此，围绕基于YOLOv11的家庭园艺植物识别系统开展设计与实现，仍然具有较强的研究价值和实践意义。

### 三、存在的问题

通过查阅相关文献可以发现，当前家庭园艺植物识别方向虽然已经有一定研究基础，但仍然存在一些比较明显的问题。

第一，数据集与实际家庭使用场景之间还有差距。很多研究使用的是公开农业数据集或者实验条件下采集的图像，这些数据通常背景相对简单、拍摄角度较规范。但家庭环境并不是这样，用户拍照时可能有阴影、反光、遮挡，也可能把植物和花盆、桌面、墙面一起拍进去。模型如果主要在较规则的数据上训练，到了真实使用时就容易出现误检和漏检[3][9]。

第二，植物类别之间相似度高，病害早期特征又不明显，这使得识别任务本身具有细粒度特点。比如一些观叶植物叶片形态接近，而病害初期可能只是局部轻微发黄或者出现小斑点，如果模型对局部细节不敏感，就很容易判断错误[1][6][14]。这说明简单地套用通用目标检测模型往往还不够，需要结合具体场景做针对性优化。

第三，系统实时性和识别精度之间仍然需要平衡。模型越大，理论上可能获得更高精度，但部署到普通设备或者移动端时，推理速度、内存占用和整体响应时间都会受到影响[5][15]。家庭园艺识别系统面对的是普通用户，如果一次识别等待太久，系统的实际使用感受就会变差。因此，如何在精度和速度之间找到合适平衡，是系统设计中必须考虑的问题。

第四，部分研究偏重算法实验，缺少完整的应用实现。有些论文在实验指标上表现较好，但没有继续完成图像上传、结果展示、用户交互和测试记录等系统部分[13][17]。对于毕业设计来说，仅有模型训练结果并不能算完整成果，还需要把算法落到系统中，形成可运行、可展示、可测试的完整作品。

第五，目前很多识别系统只给出检测框和类别名称，输出结果还不够贴近用户实际需要。对于普通家庭用户来说，他们更关心的不只是“识别出了什么”，还包括“为什么会这样”“接下来应该怎么处理”。如果系统能够在识别结果基础上补充一些简单的养护建议，实用性会更强。

### 四、课题主要内容

结合以上分析，本课题准备从数据集构建、模型训练优化、系统设计实现和测试验证四个方面展开。

首先，在数据方面，计划整合公开植物图像数据、自行收集的家庭园艺植物图像以及网络补充样本，建立一个更贴近家庭使用场景的数据集。数据处理过程中将完成图像筛选、类别划分、人工标注和训练集、测试集划分，并根据样本情况采用适当的数据增强方法，提高模型训练效果[9][16]。

其次，在模型方面，基于PyTorch框架完成YOLOv11模型训练，并结合任务书中的要求，对训练参数和检测效果进行调整。训练过程中会重点关注模型在植物种类识别和病虫害检测中的表现，同时结合Mosaic数据增强、CIoU损失函数等方法提升检测性能[10][11]。评估指标主要包括准确率、召回率、mAP以及单帧推理时间等，用来判断模型是否满足系统应用需求。

再次，在系统实现方面，计划使用Flask构建后端服务，负责接收图像、调用模型、返回识别结果；前端部分采用H5页面完成图像上传和结果展示，方便在手机端访问和使用[10][17]。在结果输出上，除了显示植物类别或病害类别外，还会加入简要的养护建议，让系统更贴近家庭用户的实际需求。

最后，在测试和文档方面，将对系统进行功能测试和性能测试，检查识别流程是否完整、页面交互是否正常、模型响应是否满足实时性要求，并整理需求分析、概要设计、模块设计、测试记录和毕业设计说明书等相关文档，保证课题成果完整。

总体来说，本课题不是单纯做一个识别模型，而是希望完成一个面向家庭园艺场景的实际应用系统。通过这一课题，可以把目标检测算法研究和系统开发结合起来，也能够体现人工智能专业毕业设计应有的工程实践特点。

## 2. 本课题要研究或解决的问题和拟采用的研究手段（途径）

本课题面向家庭园艺植物识别这一具体场景，计划设计并实现一个基于YOLOv11的植物识别系统。系统的核心目标是对常见家庭园艺植物种类以及典型病虫害症状进行识别，并通过网页端向用户展示结果和基础养护建议。整个课题既涉及模型训练，也涉及系统实现，因此需要从数据、算法和应用三个层面同时考虑。

### （一）解决的问题

1. 解决家庭园艺场景下训练数据不足、数据来源分散、样本质量不统一的问题。通过整合公开数据、自采图像和网络补充图像，构建适合本课题的数据集。
2. 解决复杂背景、光照变化、局部遮挡以及植物类别相似带来的识别精度下降问题。通过模型训练和参数优化，提高系统在真实家庭场景下的识别效果。
3. 解决模型精度和实时性之间的平衡问题。在保证识别效果的前提下，尽量降低推理时间，使系统具备较好的交互体验。
4. 解决研究只停留在算法实验层面的问题。通过完成前后端系统开发，实现图像上传、识别调用、结果返回和结果展示，形成可运行系统。
5. 解决识别结果不够直观、不够实用的问题。在输出识别结果的基础上，增加简单的养护建议，提高系统的实际使用价值。

### （二）研究手段

1. 文献研究法。查阅近5年与植物识别、病虫害检测、YOLO目标检测、轻量化部署和Web系统开发相关的文献资料，了解国内外研究现状，为课题设计提供依据。
2. 数据集构建与实验研究法。对公开数据、自采图像和网络图像进行整理、筛选和标注，划分训练集与测试集，并通过实验比较不同增强方式和训练参数的效果。
3. 模型训练与优化方法。基于PyTorch框架实现YOLOv11模型训练，结合Mosaic数据增强、CIoU损失函数和参数调节，提升检测性能，并通过实验结果不断优化模型。
4. 系统开发方法。使用Flask实现后端服务，完成图像接收、模型推理和结果返回；使用H5页面实现前端上传和结果展示，完成系统基本交互功能。
5. 测试分析法。对系统进行功能测试和性能测试，检查模型识别准确率、页面响应速度和整体运行稳定性，根据测试结果继续改进系统。

### （三）研究步骤

1. 查阅相关文献，明确课题研究背景、研究现状和技术路线，完成开题报告。
2. 完成需求分析和总体方案设计，确定系统功能模块、开发环境和实现思路。
3. 收集并整理植物图像数据，完成数据清洗、标注、增强和数据集划分。
4. 基于YOLOv11完成模型训练、参数调整和性能评估，选取合适模型用于部署。
5. 完成Flask后端和H5前端开发，实现图像上传、识别推理、结果展示和养护建议输出。
6. 对系统进行联调、测试和优化，整理开发资料、测试记录和毕业设计说明书。
