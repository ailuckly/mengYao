# 模型训练结果分析

## 1. 训练任务说明

本次实验基于项目最终训练集 `data/splits/final_v1`，采用 `YOLOv11s` 作为检测模型，对 6 类家庭园艺植物和 4 类叶部病害/异常进行联合训练。本文引用的结果来自训练批次 `colab_pro_formal_v13`。

## 2. 训练配置

- 模型权重：`yolo11s.pt`
- 训练轮次：`100 epochs`
- 批次大小：`32`
- 输入尺寸：`640`
- 优化器：`auto`
- 早停容忍：`30`
- DataLoader workers：`4`
- 保存周期：`10 epochs`
- 关闭 Mosaic：最后 `10 epochs`
- 随机种子：`42`
- 学习率策略：`cos_lr`

## 3. 关键结果

根据训练日志与 `results.csv` 统计，最佳结果出现在第 `83` 个 epoch，主要指标如下：

- Precision(B)：`0.7441`
- Recall(B)：`0.7495`
- mAP50(B)：`0.7279`
- mAP50-95(B)：`0.6754`

最后一轮 `epoch 100` 的结果略有回落：

- Precision(B)：`0.7490`
- Recall(B)：`0.7388`
- mAP50(B)：`0.7159`
- mAP50-95(B)：`0.6663`

因此，系统部署与联调采用 `best.pt` 作为默认权重，更能反映本轮实验的最佳性能。

## 4. 结果分析

从结果曲线和归一化混淆矩阵可以看出，模型表现具有以下特点：

1. 6 类植物识别效果较好，类别区分度高，整体已达到可演示水平。
2. 病害/异常类整体弱于植物类，存在明显类别不均衡和背景漏检问题。
3. `黄化/锈病/虫害类异常` 与 `白粉/霉变类病害` 是当前最薄弱的两类。
4. 指标在大约 `60-85 epoch` 之间进入平台期，继续训练到 `100 epoch` 的收益有限。

## 5. 结论

本次 `colab_pro_formal_v13` 训练结果可以作为项目的正式演示模型，适合用于：

- Web 系统联调与演示
- 论文中的主要实验结果展示
- 测试方案和功能展示章节的结果引用

## 6. 可进一步研究的方向

若后续继续提升识别效果，可优先从以下方向展开：

1. 优先优化病害类，而不是继续增加植物类样本。
2. 对病害类别做适度合并，降低类间混淆。
3. 补充病害样本或重新平衡各病害类别数量。
4. 保留当前 `YOLOv11s` 结果作为主实验，同时可补一组 `YOLOv11n` 对比实验用于论文分析。
